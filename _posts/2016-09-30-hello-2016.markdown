## 前言

Miyunluo 的 Blog 就这么上线了。

[跳过废话 ](#build)



2015 年的小学期，乱啃了一通 HTML, Bootstarp, JS, 社会实践的时候和[dusmart](https://dusmart.github.io)一起建了一个宣传网站，记得那时为了实现一个简单的弹出式窗口费了好大劲。


今年暑期本打算为自己建立个人blog，结果回家后因考驾照等种种事情延宕了。

近期偶然发现了[Github Pages](https://pages.github.com/) + [Jekyll](http://jekyllrb.com/)可以轻松解决blog问题，于是抽时间配置了环境，fork了一个model，build了自己的blog。

Git+Jekyll的优点非常明显：

* **Markdown** 带来的优雅写作体验

* 非常熟悉的 Git workflow ，**Git Commit 即 Blog Post**

* 利用 GitHub Pages 的域名和免费无限空间，不用自己折腾主机

* 如果需要自定义域名，也只需要简单改改 DNS 加个 CNAME 就好了

* Jekyll 的自定制非常容易，基本就是个模版引擎

## 正文

接下来说说配置的细节。  


配置成功后想想似乎也没有遇到什么大坑，不过一边配一边google解决问题还是花了一些时间。
主要参考[Install Jekyll on OS X El Capitan 10.11](http://digitalshore.io/how-to-install-jekyll-mac-os-x-el-capitan/)。


其中有一步需要 `sudo gem install jekyll`，而 rubygem 的源是被万里长城给墙了的……我挂着代理，没有受到影响，无法翻墙的同学可以移步[淘宝的镜像](https://ruby.taobao.org)，具体使用参见网页里的说明。

有一步在众多教程里都没有提及，如果出现报错信息`cannot load such file -- bundler (LoadError)`，需要`sudo gem install bundler`，之后`bundle install`。

最后一步build的时候，如果出现`cannot load such file -- jekyll-paginate`，那是因为fork到的modle一般都没有 Gemfile，可以自己新建一个Gemfile，或者 jekyll new 一个新project，将里面的Gemfile拷贝进去。

关于blog本身。

本地调试直接`jekyll serve --watch`，实时同步修改状态。

运用了Less CSS框架，可以直接修改Less文件以更改前端样式。

	

## 后记

Blog push到 github.io 后又遇到了一些问题，footer里的链接图标都没有加载出来，而在本地一切正常。问题见下图

![img](/img/in-post/post-blog-withno-contact.png)


---

虽然说这个东西可有可无，但是这样子看着就让我觉得不爽，晚上尝试了很久，开始以为是浏览器的锅，然而更换了浏览器依然如故，又检查了很久CSS，也没有发现问题。

早上起来后认真的想了想，知乎的“知”字显示了出来，说明应该不是页面代码的问题，所以问题因该是图标样式表的问题，要么是没有加载，要么是冲突错误。去网上搜了搜图标的class，发现用的是[Font Awesome](http://www.w3schools.com/icons/fontawesome_icons_intro.asp)，翻了翻文件，发现这个链接库在 _includes/head.html中

`<link href="http://cdn.staticfile.org/font-awesome/4.2.0/css/font-awesome.min.css" rel="stylesheet" type="text/css">`

继续搜索，发现在GithubHelp里有一篇[Securing your GitHub Pages site with HTTPS](https://help.github.com/articles/securing-your-github-pages-site-with-https/)，且给出了如下示例

![img](/img/in-post/post-blog-http2https.png)


---

问题应该是由于github采用https协议而导致链接库无法加载，解决方法就是换成https或者直接去掉，最后改为了

`<link href="//cdnjs.cloudflare.com/ajax/libs/font-awesome/4.6.3/css/font-awesome.min.css" rel="stylesheet" type="text/css">`

问题解决。
